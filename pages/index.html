Design criteria of Trust Circle Token

Anti-Whale
  • Each wallet cannot hold more than 0.0005% of the total supply equal 5,000,000,000 tokens.
  • The act of transferring money to an address that has reached 5.000.000.000 will not be allowed. Except for the case of receiving a transaction commission from trustcircle network
Anti Flash-Crash

Each transaction cannot exceed 0.0001% of the total supply equal 1,000,000,000 tokens

Eliminate infinite token mining functions

Token mining is the default function in the source code, we have removed it.

Reduce supply and distribute rewards automatically
  • 2% will be burned directly, burning will stop when the number of tokens burned is equal to 50% of the total supply.
  • The remaining 3% will be distributed to token holders according to holding ratio using a smart reward distribution algorithm called trust circle to reduce gas costs. Eligible addresses for the reward are holding a minimum of 1,000,000,000 tokens.

How trust circle give reward to you

The common way of distribution is to simply distribute the reward to all addresses simultaneously, which will increase the transaction cost for everyone if the number of addresses increases in the future as it creates a loop in the code. We solved that problem with a new algorithm called trustcircle.

Eligible addresses will be connected in a circle and there is a reward ticket that will go to each node if a transaction occurs. If there is no transaction, the bonus card will not continue to move, this ensures it will work well if the number of addresses increases rapidly, the gas cost will be solved because the loop is eliminated. and ensure a mechanism to penalize those who leave and reward those who stay.

With each transaction, the receiving and sending addresses are checked and removed or added to the bonus round. This speeds up the bonus card's faster movement to valid addresses without wasting unnecessary jumps.
