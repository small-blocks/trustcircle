Design criteria of Trust Circle Token

No whale
  • Each wallet cannot hold more than 0.0005% of the total supply equal 5,000,000,000 tokens.
  • The act of transferring money to an address that has reached 5.000.000.000 will not be allowed.
No pump and dump

Each transaction cannot exceed 0.0001% of the total supply equal 1,000,000,000 tokens

Eliminate infinite token mining functions

Token mining is the default function in the source code, we have removed it.

reduce supply and distribute rewards fairly and automatically
  • 2% will be burned directly, burning will stop when the number of tokens burned is equal to 50% of the total supply.
  • The remaining 3% will be distributed to token holders according to holding ratio using a smart reward distribution algorithm called trust circle to reduce gas costs. Eligible addresses for the reward are holding a minimum of 1,000,000,000 tokens.

How trust circle work

The first, the popular way of distribution is simply to distribute rewards to all addresses simultaneously, which will increase transaction costs for everyone if the number of addresses increases in the future because it make a looping in code.

If there is no transaction, the reward card will not keep moving, this ensures it will work well if the number of addresses increased rapidly, the gas cost will be resolved because loop is removed.

With each transaction, the receiving and sending addresses are checked and removed or added to the bonus round. This speeds up the rewards card's faster travel to valid addresses without wasting unnecessary hops.
